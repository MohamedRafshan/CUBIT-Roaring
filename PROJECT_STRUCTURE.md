## Files Modified (Minimal Changes)

### CMakeLists.txt

```cmake
# ADDED: CRoaring detection block
if (WITH_CROARING)
    find_path(CROARING_INCLUDE_DIR roaring/roaring.h ...)
    find_library(CROARING_LIBRARY NAMES roaring CRoaring ...)
    if (CROARING_INCLUDE_DIR AND CROARING_LIBRARY)
        add_definitions(-DCUBIT_HAS_CROARING=1)
        include_directories(${CROARING_INCLUDE_DIR})
        set(CROARING_LIBRARIES ${CROARING_LIBRARY})
    endif()
endif()

# ADDED: Roaring adapter files
set(ROARING_ADAPTER_FILES src/bitmap/roaring_vb.h src/bitmap/roaring_vb.cpp)

# ADDED: Link targets
add_library(bitmap STATIC ${BASE_FILES} ${FASTBIT_FILES} ${ROARING_ADAPTER_FILES} ...)
target_link_libraries(bitmap ... ${CROARING_LIBRARIES})

# ADDED: Demo target
add_executable(roaring_bitmap_demo src/bitmap/roaring_bitmap_demo.cpp)
target_link_libraries(roaring_bitmap_demo bitmap ...)
```

**All other files remain unchanged** - Zero impact on existing code paths.

```bash

  -o roaring_benchmark_windows.exe
```

```bash
# Full build with CRoaring detection

cmake ..
make -j4

# Fallback used if CRoaring unavailable
# Native CRoaring used if library found
# No breaking changes either way
```

## Deployment Paths

### Path 1: Conservative (Recommended)

1. Build existing CUBIT with adapter (no integration)
2. Benchmark both paths
3. Integrate RoaringVB into UpBit+ first
4. Run benchmarks, compare results
5. Roll out to UCB+ and other tables

### Path 2: Aggressive

1. Integrate RoaringVB into all tables immediately
2. Use CRoaring on Linux (auto-detected)
3. Fallback on other platforms
4. Full benchmark suite

### Path 3: Hybrid

1. Fallback builds everywhere (no external dependencies)
2. CRoaring builds on optimized systems
3. Transparent to application code
4. Best-case performance in production
